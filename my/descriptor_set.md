Набор дескрипторов (DescriptorSet) выделяется из заранее созданного пула дескрипторов (DescriptorPool).

Пул - это кусок памяти, размер которого нужно определить заранее. В нём нет информации, как дескрипторы распределены по наборам,
а только общее количество дескрипторов каждого типа и общее количество наборов.

Допустим у нас в шейдере 2 набора: в первом наборе буфер и текстура, а во втором текстура. Значит в пуле нужно выделить место для одного буфера,
двух текстур и двух наборов. Но это только для одного кадра и одного объекта на экране.
Объекты на экране могут иметь разную текстуру и разную матрицу трансформации - это всё увеличивает число наборов.
Кроме того обычно имеется несколько кадров одновременно (Double Buffering / Frames in Flight), значит размер пула нужно умножить
на число кадров в полете.

Поэтому размеры пулы создают с запасом, а если первый пул заканчивается, то добавляют новый.
Логичнее сделать отдельный пул для каждого типа ресурсов: для текстур, для буферов, для дескрипторов,
а когда место в одном из пулов заканчивается - создавать дополнительный этого типа.

Однако один конкретный DescriptorSet должен быть выделен целиком из ОДНОГО пула.
Нельзя собрать один DescriptorSet по кусочкам из разных пулов.

Поэтому нужно использовать подход, который называется Type-separated Descriptor Management и является шагом к Bindless Architecture (архитектуре без биндингов).

А ещё лучше использовать bindless.
